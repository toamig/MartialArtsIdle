How it scales

Spatial slots (10 producers, fixed positions)

  • Foreground: Disciples (ring around cultivator), Spirit Herb Garden (ground line), Spirit Beast Pact (beside cultivator), Ancestral Treasure (floating in front)
  • Mid layer: Meridian Furnace (left), Sect Followers (middle distance), Lunar Pavilion (right)
  • Cosmic layer: Heavenly Pillar (vertical beam), Void Conduit (purple rift), Dao Tree (filling the sky)

Tier escalation (4 visual states per producer)

  • Tier 0 (1-9 owned): 1-2 sprites, small, dim
  • Tier 1 (10-24): 3-5 sprites, normal size
  • Tier 2 (25-99): Dense, glowing, animated subtly
  • Tier 3 (100+): Mythic — bigger sprite variant, particles, dramatic glow

What's a placeholder vs final

Real (would survive): CSS layering, animations, positioning logic, ambient particles, light/glow scaling, the basic compositional layout. Placeholders: Every emoji on screen. Final versions are pixel-art sprites — the cultivator already has them; each producer needs 2-3 sprite variants for tier escalation, plus the cosmic-layer backdrops (Dao Tree, Void rift, Pillar) need bespoke artwork.

Art cost — three tiers of ambition

Minimum viable (10 sprites)

One sprite per producer. Scale, count, and CSS effects do the tier escalation. The scene fills via repetition + CSS filter passes (saturate, brightness, glow). Probably 1-2 hours in PixelLab.

Polished (20-25 sprites)

Each producer gets 2 variants — base and "epic". Plus 3-5 cosmic-layer pieces (Dao Tree silhouette, void rift texture, pillar beam). Half a day of PixelLab generation.

Full diorama (40+ sprites)

4 tier states per producer, multiple variants per tier for variety, dedicated cosmic-layer art. The Cookie-Clicker treatment. Multi-session art ask — defer.

Recommendation

Build with placeholders (this mockup's emojis) → ship → see which slots players notice → generate sprite art only for the slots that matter. Iterate from data, not from spec.
