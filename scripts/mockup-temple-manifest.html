What's a placeholder vs final art

The temple BG is a CSS stand-in here — the real game uses your existing PNG temple. Every overlay element (cushion+figure, brazier, banner, relic, beast, pavilion) is a SEPARATE small sprite you can AI-generate independently and drop in at fixed coordinates. No temple BG edits needed.

AI-art friendly

Each overlay is a small, well-defined subject on transparent BG. "Meditating disciple on a cushion", "stone brazier with flame", "hanging sect banner with yin-yang", "potted spirit herb on a stone altar" — these are easy PixelLab prompts and you can iterate one at a time.

Tier escalation per slot

Each slot has 3-4 visual states. Count thresholds (1 / 10 / 50 / 100 owned) bump the slot: more sprite instances, bigger sprite variant (sprout → tree, tiger → dragon, relic → crystal), stronger glow. Driven by CSS classes, not new art every tier.

Minimum art shopping list

10 base overlays (one per producer) + 4 "epic" variants for the producers whose Mythic state benefits from a swap (beast tiger→dragon, herb sprout→tree, relic urn→crystal, pavilion dim→lit). That's ~14 small sprites total.

Why this preserves immersion

  • The cultivator never moves; the crystal never moves; the temple itself never changes.
  • Every element added is an object a real cultivator would have in a sacred space — no flying ducks, no spaceships.
  • Light tones stay warm (gold/amber) for "in-temple" objects; cool tones (cyan, violet) only appear on cosmic-tier elements (void rift, heavenly beam, dao mural) where strange light is narratively appropriate.
  • Distant elements (pavilion, sect followers) are framed by the temple's architecture, not floating randomly.
  • The scene gets DENSER, not different. The player feels like their temple has become a place worth visiting.
