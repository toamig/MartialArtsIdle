How it scales without painting

4 sprites per producer (40 total)

Initiate (1-9) / Trained (10-24) / Master (25-99) / Immortal (100+). When the player crosses a threshold, ALL their sprites for that producer swap to the new variant simultaneously. Same as Cookie Clicker's farm columns visually upgrading at high tiers. CSS handles the threshold trigger.

Lane caps at ~20 visible units

Beyond 20 owned, the lane shows a +N overflow indicator on the right edge. The visual stays scannable; the count carries the "I own a lot" feeling. Mythic-tier lanes override the overflow with a colored badge so the player sees their "completion" status.

No backdrop. No composition.

Each lane is a horizontal strip. Layout is pure flexbox. The unit sprites are independent on transparent BGs — drop them in fixed positions. Zero painted scenery to worry about.

Tier escalation also via CSS

The leader sprite gets an inset glow + outer shadow per tier (Bronze warm / Silver pale / Gold radiant / Mythic violet shimmer). Same emoji here as placeholder; final art uses the tier-variant sprites. Tier badge ribbon in the meta row tells the player exactly where they are.

Where this lives in the game

Two natural homes:

  • CultivationScreen — top section: split the screen. Top half shows these lanes (your "forces"); bottom half stays the buy list. Player sees what they've built WHILE shopping for more.
  • New "Domain" tab: separate navigation entry. Player taps it to admire their accumulated forces. Cleaner but requires extra nav tap to see.

I'd recommend CultivationScreen top section — colocated with buying = stronger feedback loop.

Art shopping list — what to generate first

Initiate10 sprites — basic, single-color silhouette per producer
Trained10 sprites — slightly more detail, mild armor/glow
Master10 sprites — distinct visual upgrade (different posture/gear/structure)
Immortal10 sprites — full glow, ornate, distinct color signature

40 sprites total. Each is 32×32 or 48×48, transparent BG. Easy PixelLab prompts: "meditating disciple pixel art, transparent, 32x32". You can generate one tier-set at a time (10 sprites per session) and iterate quickly. Worst case you ship with 1 sprite per producer and let the CSS filter do all the tier escalation — that's 10 sprites and the game still feels like CC.

What's a placeholder vs final

Placeholders: every emoji. The Master tier "swap" to ⚔️ and the Mythic tier swap to ✨ are illustrating the concept — final art keeps the disciple identity but with different gear/glow across tiers. Real (would survive): the lane layout, sprite stacking logic, count cap + overflow indicator, tier badges, CSS glow escalation, breathing animation, hover state.
